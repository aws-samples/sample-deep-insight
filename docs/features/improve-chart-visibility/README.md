# Improve Chart Visibility & Data Validation

Initiative covering two related quality dimensions of the DOCX reports
generated by Deep Insight (managed-agentcore + self-hosted):

1. **Data validation** — every body claim traces to a verified
   computation, no provisional values reach the reader.
2. **Chart quality** — every chart is legible at DOCX display size,
   no clipped text, no overlapping annotations, vector-rendered.

## Approach

Pure prompt-engineering changes — no code rebuild required.
Improvements landed as additive rules in the Coder / Reporter /
Validator agent prompts plus a `svg_docx.py` utility that the
Reporter writes verbatim into the Fargate executor sandbox at runtime
(Step 0a pattern, since the sandbox cannot import from the Runtime
container).

## Part 1 — Data Validation

### Pipeline

```
Coder ─ track_calculation() ─→ citations.json
                                     │
Validator ─ independent recompute ─→ verification_status (verified|needs_review)
                                     │
Reporter ─ quote only verified ─→ body + references
          │
          └─ audit_body_citations() scans for orphan claims
```

### What Coder tracks (`<citation_coverage>` rule)

Track every numeric value whose calculation method matters — values
where a reader might reasonably ask "how was this computed?" or where
a different methodology could produce a different answer.

**Test for analytical-vs-descriptive**: if computing the value
requires methodology choices (which threshold, which weighting, which
subset, which time window), it is an analytical finding — track it.
If the only "calculation" is enumerating what is already in the data,
it describes input scope — state in prose without tracking.

Default to over-tracking analytical values, not descriptive ones.

### How Reporter cites (`<numerical_claim_discipline>` rule)

The body must quote only verified analytical claims. Every ≥3-digit
number representing an analytical finding must trace to citations.json
— EITHER directly via a [N] marker, OR transparently via arithmetic of
cited values quoted nearby.

The value and its [N] marker must always travel together — never a
marker without a value, never a tracked finding without its marker.
Numbers describing input data scope (row counts, date ranges) do NOT
need [N] markers and SHOULD NOT carry them.

**ENFORCEMENT**: `audit_body_citations(doc)` is called in
reporter_final.py before the references section. Skipping it ships
unverified body claims — a process violation, not a stylistic choice.

### needs_review handling (`<value_status_resolution>` rule)

Body prose, table cells, and references must contain only `verified`
citations. Values with `verification_status='needs_review'` are
EXCLUDED from the published report — they remain in citations.json
and the audit log for ops review, but no value, marker, or label
reaches the reader-facing DOCX.

If a section relies on a needs_review value, restructure to use only
verified values, or omit the claim. Internal QA artifacts must not
appear in the published artifact.

### References format

Each reference entry shows `description + formula + value` plainly.
Do NOT prefix the value with validation-context labels regardless of
citation status — even when the value matches what Validator
confirmed, such labels imply ongoing QA context to the reader and
turn a clean reference into technical noise. The value alone, with
its description and formula, is enough.

## Part 2 — Chart Quality

### Text legibility (rcParams baseline)
- `text.color` / `axes.labelcolor` / tick / title color: `#000000`
- weight: `bold` for axes labels, titles, ticks, annotations
- baseline `font.size=12`, `axes.titlesize=16`, ticks/legend `=11`
  (calibrated for ~5.5" display width)

### Chart type restrictions
- **Pie chart**: forbidden as standalone — default to horizontal bar
  for category distribution. Permitted ONLY when paired (≥2 side-by-
  side) where part-whole comparison is the central message.
- **3D plots / parallel coordinates / word clouds**: forbidden entirely
  (illegible at DOCX scale).
- Density antipatterns: scatter forbidden at N≥500 (use hexbin),
  Korean labels default to horizontal bars, time-series >15 series
  → heatmap.

### Annotation rules
- Bar charts (≤20 bars total): label every bar with compact format.
  >20 bars → top-N (5-10) only.
- Single-line time-series (N≤15): label every point. N>15 → first /
  last / min / max with dedupe.
- Multi-line (2-4 series): direct label at last x-position. Multi-
  line (5+): legend only.
- adjustText pre-pad ALL four sides by 6% before `adjust_text()`.
- After adjust_text(), verify each text's position; hide labels that
  escaped the padded bounds (`set_visible(False)`). `set_clip_on(True)`
  is unreliable because `ax.annotate(..., bbox=...)` draws its bbox
  patch through a separate renderer path.
- Quadrant-balanced selection when ≥6 labels (groupby quadrant, top-N
  per quadrant) to prevent same-quadrant clustering.

### Reference lines
- Annotate axhline/axvline via `ax.text()` with `transform=
  ax.get_yaxis_transform()` (or x equivalent) — NEVER `fig.text()`.
- If reference value falls outside the visible y range, OMIT the
  reference entirely — do NOT draw axhline AND do NOT add positional
  text at y=value. With `bbox_inches='tight'`, any text artist outside
  ylim causes the saved figure to expand vertically, leaving the
  chart compressed and the label floating above the chart frame.
- Same logic for axvline / x-axis references.

### Layout rules
- Multi-panel grid avoidance: 2×2 grids forbidden, prefer single panel
  + twin axes when each subplot has 12+ x-ticks.
- ylim padding: `*1.15` default for bar/stacked/histogram, `*1.18`
  with top labels, `*1.20` with reference lines.
- Cluster-concentrated figsize boost: when ≥4 annotations share one
  quadrant in scatter/bubble, raise figsize height by 30%.

### SVG dual-embed (vector rendering)
- `svg_docx.py` finalizes DOCX by injecting OOXML `asvg:svgBlip`
  extension on every embedded PNG that has a matching `.svg` sibling.
- Word 2016+ / LibreOffice 6.0+ render SVG natively; older viewers
  use PNG fallback (PNG is required by OOXML schema as primary image).
- `svg.fonttype='path'` bakes Korean glyphs as Bezier paths — fully
  font-independent SVG.
- MANDATORY POST-PROCESSING block in reporter_final.py with
  `DO NOT REMOVE` marker resists LLM mutation of the final embed step.

### Korean rendering
- NanumGothic baked into PNG via matplotlib font config.
- SVG glyphs rendered as paths (no `<text>` elements that depend on
  reader-side font availability).

## General Principles

- **Prefer general principles over symptom-specific anchors.** Prompt
  rules state the principle in 1-2 lines; regression citations and
  per-case enumeration are avoided (they fail to transfer to similar-
  but-not-identical cases).
- **Variance vs. signal.** Before treating a metric difference as a
  regression, check whether it reflects natural LLM variance.
- **Defense in depth.** Coder and Reporter each enforce data-validation
  rules independently — Coder decides what to track, Reporter decides
  what to cite. Both layers must be self-contained.

## Verification

Multiple production runs across diverse datasets verified each rule
visually (chart-by-chart review) and programmatically (audit count,
marker presence, references format check).
